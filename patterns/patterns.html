Pattern Library

Reusable patterns extracted from the five core pages. Each category page shows variants side by side for review and approval before WordPress implementation.

About this library

These patterns are extracted directly from the five finalised pages — home, services, blog, post, and contact. No new design language is introduced here. Each pattern is a named, portable slice of what already exists. The naming convention is pattern-[category]-[variant], matching the class names in the source files.

Review each category page in turn. Flag anything that deviates from the established system. After approval, these become the pattern register for the WordPress block theme and companion plugin.
